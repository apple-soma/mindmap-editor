MindMapエディターの画面仕様
シンプルかつ直感的に使えるMindMapエディターのUI/UXを設計します。

1. 画面構成
① メインエリア（マインドマップ描画エリア）
役割: ノード（トピック）を作成・編集・削除・移動できる。
仕様:
ノード（円や長方形）を配置し、ドラッグで移動可能。
Tabキーで子ノードを追加。
Enterキーで同じ階層のノードを追加。
Deleteキーで選択したノードを削除。
ノード間の関連線（エッジ）を自動で描画。
② ツールバー（画面上部）
役割: 操作ボタンを配置。
仕様:
新規作成: マインドマップをリセットする。
データの保存・読み込み
ローカルストレージ / クラウド（Firebase, Supabase）保存対応。
エクスポート
JSON、PNG、PDF形式でのエクスポート。
スタイル変更
ノードの色・フォント・サイズの変更。
③ プロパティパネル（画面右側）
役割: ノードの詳細設定。
仕様:
選択中のノードのテキスト編集。
フォントサイズ・色のカスタマイズ。
ノードの削除。
④ フッター（画面下部）
役割: ショートカットキー一覧を表示。
仕様:
Tab → 子ノードを追加
Enter → 同じ階層にノード追加
Delete → ノード削除
Ctrl + S → 保存
Ctrl + Z / Ctrl + Y → Undo / Redo
2. 画面デザインのイメージ
メインエリア中央にルートノード（親ノード）を配置。
ツールバー・プロパティパネル・フッターを固定し、視認性を向上。
ドラッグ&ドロップでノードの配置変更可能。
3. 機能の拡張性
リアルタイムコラボレーション（Firebase / WebSocket）
AIによるノード整理・自動補完
ダークモード対応



		Ctrl+Y で Redo（やり直し）を追加
		最大履歴数を設定し、メモリ使用量を最適化
		履歴リストを UI に表示し、手動で選択できるようにする
履歴リストをリスト形式（ドロップダウンなど）に整理
履歴の時間や変更内容を表示
Redo の最大履歴数も制限
